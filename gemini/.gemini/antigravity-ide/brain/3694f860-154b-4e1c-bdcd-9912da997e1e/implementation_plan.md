## 2. 미션 요구사항 대비 취약점 분석

미션.txt를 한 줄 한 줄 대조하면서, 현재 기획에서 놓치기 쉬운 부분을 3단계로 분류했습니다.

### 🔴 치명적 (이것을 빠뜨리면 미션 불합격)

> [!CAUTION]
> **1. "실제 동작하는 워크플로우"만 인정 (미션 L86)**
> 설계 문서만 제출하면 불합격입니다. Make와 Zapier 모두에서 **실제로 폼을 제출하고, 시트에 데이터가 들어가고, 슬랙에 알림이 온 증거(스크린샷)**가 반드시 있어야 합니다.

> [!CAUTION]
> **2. 각 분기 경로가 실제로 1회 이상 실행된 결과 필요 (미션 L61)**
> 3-Way Router를 설계했으므로, 분기 A(고액), B(소액), C(예외) 모두 최소 1번씩 실행된 증거가 필요합니다. 테스트 데이터 3종을 반드시 준비하세요.

> [!CAUTION]
> **3. 프로젝트 2가 통째로 빠져 있음 (미션 L39~L44)**
> 현재 기획은 프로젝트 1에만 집중되어 있습니다. 프로젝트 2(자유 주제)도 별도로 만들어 제출해야 합니다. 프로젝트 2에서는 Trigger 자동 실행이 필수입니다(미션 L87).

### 🟡 주의 (빠뜨리면 감점 또는 미흡 평가)

> [!WARNING]
> **4. 비교 분석 보고서 필수 포함 항목 (미션 L65~L70)**
> 보고서에 반드시 들어가야 하는 5개 항목:
> 1. 사용한 도구 이름
> 2. 구현 과정 요약
> 3. 최소 5개 이상의 비교 항목
> 4. 각 도구의 장단점 정리
> 5. 어떤 상황에서 적합한지 의견
>
> 현재 `AI_자동화_도구_비교_분석.md`는 조사 문서이지 제출용 보고서가 아닙니다. 위 5개를 모두 담은 별도 보고서를 만들어야 합니다.

> [!WARNING]
> **5. 보안 마스킹 (미션 L88~L90)**
> 스크린샷에 API Key, 이메일 주소, 토큰 등이 보이면 안 됩니다. 캡처 후 반드시 가림 처리(모자이크/*** 처리)를 해야 합니다.

## 3. 단계별 실행 로드맵

천천히 한 단계씩 진행합니다. 각 단계를 끝낸 뒤 다음 단계를 요청해 주세요.

### 📌 1단계: 데이터 소스 준비 (예상 15분)
구글 폼 1개, 구글 시트 1개를 만듭니다.
- 구글 폼: 질문 1개 (지출 메모, 장문형)
- 구글 시트: 헤더 6열 (타임스탬프, 원본 메모, 카테고리, 금액, 요약, 특이사항)

### 📌 2단계: Make.com 워크플로우 조립 (예상 40분)
[walkthrough.md](file:///C:/Users/Codyssey/.gemini/antigravity-ide/brain/3694f860-154b-4e1c-bdcd-9912da997e1e/walkthrough.md)에 정리된 설정값을 그대로 복사-붙여넣기하며 조립합니다.
- 트리거 → OpenAI → Router(3분기) → 시트/슬랙
- 3가지 테스트 데이터 입력 → 각 분기 실행 확인 → 스크린샷 캡처

### 📌 3단계: Zapier 워크플로우 조립 (예상 30분)
Make와 동일한 로직을 Zapier Paths로 구현합니다.
- 같은 3가지 테스트 → 실행 확인 → 스크린샷 캡처

### 📌 4단계: 프로젝트 1 보고서 작성 (예상 30분)
스크린샷과 비교 분석을 합쳐서 최종 마크다운 보고서를 만듭니다.
- 이 단계에서 제가 보고서 틀(5개 비교 항목 표 포함)을 생성해 드립니다.

### 📌 5단계: 프로젝트 2 설계 및 구현 (예상 60분)
자유 주제를 새로 기획하고, 도구 1개로 구현합니다.
- 이 단계에서 제가 주제 후보 제안 + 워크플로우 설계 문서를 만들어 드립니다.
